nvim-in-browser — Configuration

Edit your multi-file Neovim config and install pure-Lua plugins — it all runs in a WASM sandbox.

This runs in a browser sandbox. Pure-Lua/Vimscript config, keymaps, options, and pure-Lua plugins work. Config that spawns processes or uses the network (LSP servers, git, telescope+ripgrep, plugin managers that clone repos) will NOT work. Multi-file config editing and pure-Lua plugins install below.
    Installs into pack/plugins/start and auto-loads on boot. Plugins that spawn processes or use the network (LSP, Telescope+ripgrep, Treesitter parsers, Mason, plugin managers) will not work.
    …or upload a plugin folder from disk
    Raises the API rate limit (60/hr per-IP → 5,000/hr) and lets you install from your private repos. A fine-grained, read-only token is enough (add Contents: read only for the private repos you'll install). Stored in a separate local database, never written into your config or the editor, sent only to GitHub over HTTPS.
